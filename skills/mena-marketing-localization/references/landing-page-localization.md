# Landing Page Localization

Use this reference for Arabic, English, French, or bilingual landing pages targeting Arab/MENA markets.

## Minimum Localization Fields

| Area | Check |
| --- | --- |
| Country | Country and city/region where relevant. |
| Segment | B2B, B2C, ecommerce, SaaS, app, marketplace, public sector, or regulated. |
| Language | MSA, dialect, English, French, bilingual, or unknown. |
| Currency | Display local currency or explain why not. |
| Payments | Payment method notes must be source-backed or marked unknown. |
| Delivery/returns | State local delivery, return, refund, and support terms if relevant. |
| Business identity | Company name, legal entity, address, tax/VAT where needed, and customer support channel. |
| Privacy | Privacy notice, opt-out, cookie/retargeting consent, and data minimization. |
| Claims | Substantiate pricing, savings, rankings, testimonials, and category claims. |

## Scoped Notes

- Country: UAE. Segment: ecommerce/marketplace. Channel: landing page. Confidence: B. Evidence: EZDubai/Euromonitor release mentions payment, logistics, and returns as ecommerce factors. Agent implication: localize these trust fields and avoid category conversion claims without category evidence.
- Country: Morocco. Segment: SaaS/ecommerce/public services. Channel: SEO landing page. Confidence: needs native review. Evidence: DataReportal and Morocco Digital 2030 support digital context, but not language priority. Agent implication: test Arabic/French/English keyword intent with local SEO review.
- Country: Multi-market Arab/MENA. Segment: all. Channel: TikTok ads to landing page. Confidence: A for mechanics. Evidence: TikTok policy docs require ad/landing relevance and acceptable language/currency for target market. Agent implication: check ad, landing language, currency, product claims, and destination functionality together.

## Arabic And RTL QA

- Use correct `dir` and language tags for Arabic or mixed Arabic-English UI.
- Keep numbers, phone numbers, currency, and product names readable in RTL layouts.
- Test mobile page width, forms, checkout, modals, and sticky bars in Arabic.
- Do not final-publish dialect copy without native review.

## Approval Gate

Before production launch, verify:

- Country, segment, channel, language, and offer are stated.
- All visible claims are source-backed or removed.
- Privacy, consent, retargeting, and analytics are reviewed.
- Platform landing-page policies are checked for paid traffic.
- Native review is complete for public Arabic/French/localized copy.
