# Support Tone

Use this file when drafting Arabic, bilingual, or customer-facing support copy.

## Default Language

- Default to Modern Standard Arabic for cross-market support.
- If the customer writes Arabic, answer in Arabic.
- If the customer mixes Arabic and English, answer Arabic first and add a short English mirror only when helpful.
- If the brand asks for dialect, mark the copy `native-review-needed` unless a country-specific native reviewer has approved that tone for the channel and use case.

## Tone Modes

| Mode | Use for | Notes |
| --- | --- | --- |
| Formal MSA | Fintech, tax, identity, legal-sensitive refunds, B2B, government, serious complaints | Keep short, precise, and non-casual. |
| Semi-formal MSA | Retail, ecommerce, delivery, subscription, account help | Warm but not slangy; no jokes in complaints. |
| Dialect | Country-specific retail support only after review | Always mark `native-review-needed` until reviewed. |
| Public social | Instagram comments, public posts | Acknowledge once and move to private or secure channel. |

## Style Rules

- Start with acknowledgement, then status, then next step.
- Avoid blame: say "لم تكتمل العملية" instead of blaming the customer's input.
- Use empathy without admitting liability when facts are unclear: "نتفهم انزعاجك".
- Use apology when the business caused or likely caused the disruption: "نعتذر عن الإزعاج".
- Avoid over-promising. Use "سنراجع الحالة" instead of a refund or delivery promise unless policy supports it.
- Keep mobile messages short enough for WhatsApp.
- Use consistent numerals based on brand locale.

## Native Review Flags

Flag `native-review-needed` for:

- Dialect, Arabizi, idioms, humor, slogans, or affectionate terms.
- Public customer-facing Arabic that will be reused as an approved macro.
- Complaints, refunds, identity, fintech, tax, or legal-sensitive support copy.
- Country-specific terms for delivery, wallet, invoice, cancellation, or complaint.
