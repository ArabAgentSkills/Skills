# Arab Market Context Integration Checklist

Use this checklist when country context affects implementation handoffs.

## Answer Shape

1. Files read.
2. Source-backed country facts.
3. Planning hypotheses.
4. Unknowns and review needs.
5. Skill handoffs.
6. Validation steps.
7. Production launch gate.

## Checks

- Confirm country, audience, channel, and intended use.
- Load the relevant country file and source/confidence references.
- Do not turn country context into vendor API instructions.
- Route implementation to the relevant skill and vendor source.
- Keep sandbox/draft work separate from production/live work.
- Protect customer, payment, identity, tax, survey, reviewer, and support data.

## Production launch gate

Before any live implementation, sending, publishing, payment, refund, identity, tax, audience upload, or customer-data action:

- Recheck source freshness.
- Complete native or expert review when required.
- Validate vendor docs and sandbox behavior in the domain skill.
- Get explicit human approval using the relevant live-action verb.
