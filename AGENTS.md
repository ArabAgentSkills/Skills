# Arab Agent Skills Public Repo Instructions

```text
ARAB AGENT SKILLS
source-backed regional API skills for AI agents
```

- Keep `SKILL.md` files short and procedural.
- Put vendor facts in `skills/*/vendors/*.md`, not in routing instructions.
- Do not invent endpoints, credentials, SDK support, pricing, or compliance claims.
- Prefer official docs, developer portals, OpenAPI specs, Postman collections, and official SDK repos.
- Mark gated docs, unknown access, and uncertain claims honestly.
- Never commit API keys, customer data, private docs, or production credentials.
- Keep sandbox and production instructions separate for payments, identity, tax, payouts, refunds, and open banking.
- Update eval prompts and expected behavior files when a skill changes.
